# START HERE — как продолжить проект Bozor

## 1. Открой эту папку в VS Code
File → Open Folder → выбери папку `bozor-marketplace`.

## 2. Убери ошибку обновления Claude Code (если есть)
Закрой лишние окна VS Code и терминалы с Claude Code, оставь одно окно.
Ошибка `claude.exe in use` на проект не влияет.

## 3. Посмотри быстрый предпросмотр (по желанию)
Открой `index.html` двойным кликом в браузере — но сначала впиши anon-ключ:
найди строку `const SB_KEY='__SB_ANON_KEY__';` и подставь свой
**anon public** ключ из Supabase → Project Settings → API.

## 4. Запусти Claude Code в этой папке и вставь промпт
Промпт ниже. Claude Code сам прочитает CLAUDE.md и продолжит.

---

Ты в новой папке проекта "Bozor" — маркетплейс объявлений для Узбекистана.
Прочитай CLAUDE.md в корне — там весь контекст (Supabase уже развёрнут,
схема и данные готовы, GitHub-репозиторий создан). dilmatch не трогай.

Сделай:
1. Предложи структуру проекта (рекомендую Next.js + @supabase/supabase-js,
   с заделом под Telegram Mini App). Дождись моего "ок".
2. Собери каркас, подключи Supabase (env в .env.local, .gitignore уже есть).
3. Главный экран: лента смартфонов из таблицы listings (продавец, рейтинг),
   фильтры бренд/цена/состояние, карточка объявления — по образцу index.html.
4. git init → коммит → push в https://github.com/oyatbozo-oss/bozor-marketplace.git (main).
5. Деплой на Vercel как НОВЫЙ проект (npx vercel), не трогая dilmatch.

Сначала покажи план и структуру папок, дождись подтверждения, потом делай.

---

## Важно
- anon public ключ — публичный, можно в .env.local. service_role — секретный, не коммить.
- Всё в рамках бесплатных планов.
